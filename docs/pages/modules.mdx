# Modules

## v0.8

### Alchemy

- **[SingleSignerValidationModule](https://github.com/alchemyplatform/modular-account/blob/develop/src/modules/validation/SingleSignerValidationModule.sol)**: Enables validation for a single signer (EOA or contract).
- **[WebAuthnValidationModule](https://github.com/alchemyplatform/modular-account/blob/develop/src/modules/validation/WebAuthnValidationModule.sol)**: Enables validation for passkey signers.
- **[AllowlistModule](https://github.com/alchemyplatform/modular-account/blob/develop/src/modules/permissions/AllowlistModule.sol)**: Enforces ERC-20 spend limits and address/selector allowlists.
- **[NativeTokenLimitModule](https://github.com/alchemyplatform/modular-account/blob/develop/src/modules/permissions/NativeTokenLimitModule.sol)**: Enforces native token spend limits.
- **[PaymasterGuardModule](https://github.com/alchemyplatform/modular-account/blob/develop/src/modules/permissions/PaymasterGuardModule.sol)**: Enforces use of a specific paymaster.
- **[TimeRangeModule](https://github.com/alchemyplatform/modular-account/blob/develop/src/modules/permissions/TimeRangeModule.sol)**: Enforces time ranges for a given entity.

### Circle

- **[SingleSignerValidationModule](https://github.com/circlefin/buidl-wallet-contracts/blob/master/src/msca/6900/v0.8/modules/validation/SingleSignerValidationModule.sol)**: Enables validation for a single signer (EOA or contract). Uses a different signature format.
- **[WeightedMultisigValidationModule](https://github.com/circlefin/buidl-wallet-contracts/blob/master/src/msca/6900/v0.8/modules/multisig/WeightedMultisigValidationModule.sol)**: Enables validation via a weighted multisig.
- **[ColdStorageAddressBookModule](https://github.com/circlefin/buidl-wallet-contracts/blob/master/src/msca/6900/v0.8/modules/addressbook/ColdStorageAddressBookModule.sol)**: An enhanced version of `AddressBookModule`, incorporating additional limitations on target contracts.

### Axiom

- **[KeystoreValidator](https://github.com/axiom-crypto/keystore-periphery/blob/main/contracts/src/KeystoreValidator.sol)**: Facilitates keystore chain usage for modular accounts.

### Sarkazein

- **[GPGValidationModule](https://github.com/sumitvekariya/gpg-modular-account)** Allopws a GPG public/private keypair to validate transactions and signatures for modular smart accounts.

## v0.7

### Alchemy

- **[SessionKeyPlugin](https://github.com/alchemyplatform/modular-account/tree/v1.0.x/src/plugins/session)**: Session keys can be added to conduct various actions on behalf of the modular account under preset rules. For v0.8, this functionality has been redesigned to make the permissions more modular.
- **[MultiOwnerPlugin](https://github.com/alchemyplatform/modular-account/tree/v1.0.x/src/plugins/owner)**: One or more EOA accounts or ERC-1271 compliant contracts can be owners of the Modular Smart Contract Account.
- **[ColdStoragePlugin](https://github.com/alchemyplatform/aa-virtual-cold-storage)**: An unaudited plugin to demonstrate plugin capabilities. Allows the modular account to secure NFTs using a separate storage key, effectively placing it in cold storage.

### Alchemy x Maple Finance

- **[MultiSigPlugin](https://github.com/alchemyplatform/multisig-plugin/blob/v1.0.x/src/MultisigPlugin.sol)**: This plugin adds a k of n threshold ownership scheme to a ERC6900 smart contract account.

### Exactly

- **[WebauthnOwnerPlugin](https://github.com/exactly/webauthn-owner-plugin/blob/main/src/WebauthnOwnerPlugin.sol)**: One or more EOA accounts, ERC-1271 compliant contracts, or P256 passkeys can be owners of an account.
- **[ExaPlugin](https://github.com/exactly/mobile/blob/main/contracts/src/ExaPlugin.sol)**: Plugin that enables seamless UX for Exa App users using modular accounts.

### Circle

- **[SingleOwnerPlugin](https://github.com/circlefin/buidl-wallet-contracts/blob/v2.1.0/src/msca/6900/v0.7/plugins/v1_0_0/acl/SingleOwnerPlugin.sol)**: This plugin allows MSCA to be owned by an EOA or another smart contract (which supports 1271).
- **[DefaultTokenCallbackPlugin](https://github.com/circlefin/buidl-wallet-contracts/blob/v2.1.0/src/msca/6900/v0.7/plugins/v1_0_0/utility/DefaultTokenCallbackPlugin.sol)**: Default token callback handler plugin. Similar to DefaultCallbackHandler. This plugin allows MSCA to receive tokens such as 721, 1155 and 777.
- **[WeightedWebauthnMultisigPlugin](https://github.com/circlefin/buidl-wallet-contracts/blob/v2.1.0/src/msca/6900/v0.7/plugins/v1_0_0/multisig/WeightedWebauthnMultisigPlugin.sol)**: This plugin adds a weighted threshold ownership scheme to a ERC6900 smart contract account.
- **[AddressBookPlugin](https://github.com/circlefin/buidl-wallet-contracts/blob/v2.1.0/src/msca/6900/v0.7/plugins/v1_0_0/addressbook/AddressBookPlugin.sol)**: This plugin allows MSCA to check if the destination address is allowed to receive assets like native token, ERC20 tokens, etc.
- **[ColdStorageAddressBookPlugin](https://github.com/circlefin/buidl-wallet-contracts/blob/v2.1.0/src/msca/6900/v0.7/plugins/v1_0_0/addressbook/ColdStorageAddressBookPlugin.sol)**: This plugin serves as an enhanced version of the AddressBookPlugin, incorporating additional limitations on target contracts.

### Libree

- **[SubscriptionPlugin](https://github.com/Libree/subscription-plugin)**: Users can subscribe to services and set up recurring subscription payments.

### Locker Labs

- **[SavingsModule](https://github.com/locker-labs/savings-module)**: Save a little to an external savings account every time the smart account performs an ERC-20 transfer.
